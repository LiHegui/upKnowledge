# Canvas 大数据渲染 — LiDAR 点云实战

> **场景**：后端推 10 万 ~ 100 万个 LiDAR 点云数据，前端用 **Canvas 2D**（不上 WebGL）渲染，要求 60fps 可缩放/拖动/悬停查询。

---

## 一、问题本质

| 维度 | 普通图表 | LiDAR 点云（canvas 2D 渲染） |
|------|---------|------------------------------|
| 数据量 | 几百~几千 | **10w+** |
| 渲染方式 | `arc` / `fillRect` 一点点画 | **像素级直接写入** |
| 主线程压力 | 一次画完无所谓 | **必须丢 Worker** |
| 交互压力 | 几乎没有 | 拖动/缩放/悬停 60fps |
| 瓶颈 | 渲染 | **数据传输 + 像素写入** |

**核心反转认知**：
> **10w 点对屏幕来说不算多 —— 1920px 屏幕也就 1920 列像素，画 10w 个点最多落到 200w 个像素。瓶颈不是"画"，是"怎么把每个点高效变成 1 个像素写入"。**

---

## 二、整体架构（端到端）

```
┌─────────────┐  二进制流  ┌──────────────┐  postMessage  ┌────────────────┐
│  后端       │ ─────────> │  主线程      │ ────────────> │  Worker        │
│  PCD/LAS    │  ArrayBuf  │  WebSocket   │  (零拷贝)     │  (OffscreenCvs)│
│  转 Float32 │            │  接收/解析   │               │  渲染          │
└─────────────┘            └──────────────┘               └────────────────┘
                                  ▲                                │
                                  │ 视口/交互                       │ 像素数据
                                  │ postMessage                    │
                                  └────────────────────────────────┘
                                  
┌──────────────────────────────────────────────────────────────────┐
│                   DOM 上 3 层 Canvas                              │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ Layer 1: 背景层 (网格/坐标轴) - 静态，一次画完           │    │
│  │ Layer 2: 点云层 (OffscreenCanvas) - Worker 渲染          │    │
│  │ Layer 3: 交互层 (悬停高亮/选区) - 主线程 60fps           │    │
│  └─────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────┘
```

## 三、五层优化策略（按性价比排序）

### 1. ⭐⭐⭐ 用 `ImageData` 直接操作像素（最猛）

**别用 `fillRect` / `arc` 一个个画，太慢**。直接操作底层像素数组 → **10-50 倍提升**。

```js
function renderPointsByImageData(points, ctx, w, h) {
  const imageData = ctx.createImageData(w, h)
  const data = imageData.data   // Uint8ClampedArray，长度 w*h*4 (RGBA)

  for (let i = 0; i < points.length; i++) {
    const p = points[i]
    const x = Math.round(p.x)
    const y = Math.round(p.y)
    if (x < 0 || x >= w || y < 0 || y >= h) continue

    const idx = (y * w + x) * 4
    data[idx]     = p.r ?? 255   // R
    data[idx + 1] = p.g ?? 255   // G
    data[idx + 2] = p.b ?? 255   // B
    data[idx + 3] = 255          // A
  }

  ctx.putImageData(imageData, 0, 0)
}
```

**为啥快**：
- ❌ `ctx.fillRect(x, y, 1, 1)` 走完整 canvas 状态机，每次有上下文切换开销
- ✅ `ImageData` 直接写内存，最后一次性 `putImageData` 提交

**实测**：10w 点从 200ms 降到 < 10ms。

---

### 2. ⭐⭐⭐ Web Worker + OffscreenCanvas

主线程不能卡 → 整个渲染丢 Worker 里：

```js
// main.js
const canvas = document.getElementById('cloud')
const offscreen = canvas.transferControlToOffscreen()
const worker = new Worker('renderer.js')
worker.postMessage(
  { canvas: offscreen, points },
  [offscreen, points.buffer]    // ← Transferable，零拷贝
)
```

```js
// renderer.js
self.onmessage = ({ data: { canvas, points } }) => {
  const ctx = canvas.getContext('2d')
  renderPointsByImageData(points, ctx, canvas.width, canvas.height)
}
```

**关键点**：
- **`OffscreenCanvas`** —— Worker 直接渲染（Chrome/Edge 全支持，Safari 16.4+）
- **`Transferable Objects`** —— 传 `ArrayBuffer` 不复制，10MB 数据 0ms 传过去
- **降级方案**：Safari 旧版不支持 OffscreenCanvas → 主线程渲染 + `setTimeout(render, 0)` 分帧

---

### 3. ⭐⭐⭐ 数据用 `Float32Array`，别用对象数组

```js
// ❌ 慢：每个点都是对象，遍历有属性访问开销
const points = [{ x: 1, y: 2, intensity: 0.5 }, ...]

// ✅ 快：扁平 TypedArray，CPU 缓存友好
const positions = new Float32Array(n * 3)  // [x,y,intensity, x,y,intensity, ...]

for (let i = 0; i < n; i++) {
  const offset = i * 3
  const x = positions[offset]
  const y = positions[offset + 1]
  const intensity = positions[offset + 2]
}
```

**优势**：
- 内存连续，CPU 缓存命中率高（2-3 倍提升）
- 可以零拷贝传给 Worker
- 后端直接发二进制（不用 JSON 序列化）

**配套**：后端约定二进制协议（如 PCD/LAS 简化版），前端 `fetch().arrayBuffer()` 收。

### 4. ⭐⭐ 分层 canvas（静态 + 动态）

```html
<div style="position: relative">
  <canvas id="bg"     style="position: absolute"></canvas>  <!-- 网格/坐标轴，一次画完 -->
  <canvas id="cloud"  style="position: absolute"></canvas>  <!-- 点云，需要更新 -->
  <canvas id="hover"  style="position: absolute"></canvas>  <!-- 悬停高亮，频繁更新 -->
</div>
```

**好处**：交互层（悬停）每帧清空重画，不影响下层 10w 点。

### 5. ⭐⭐ 视口剔除 + LOD（Level of Detail）

```js
// 只画屏幕能看到的点
function isInViewport(sx, sy, w, h) {
  return sx >= 0 && sx < w && sy >= 0 && sy < h
}

// LOD：缩得很小时降采样
function getLODStep(zoom) {
  if (zoom < 0.5) return 10   // 每 10 个点取 1 个
  if (zoom < 1)   return 3
  return 1                     // 不抽样
}
```

### 6. ⭐⭐ 空间索引（RBush / QuadTree）—— 悬停查询

10w 点鼠标悬停做最近邻查询，**O(n) 遍历会卡死**。用四叉树：

```js
import RBush from 'rbush'

const tree = new RBush()
tree.load(points.map(p => ({
  minX: p.x, minY: p.y, maxX: p.x, maxY: p.y, data: p,
})))

// 鼠标位置附近查询：O(log n)
canvas.onmousemove = e => {
  const hits = tree.search({
    minX: e.x - 5, minY: e.y - 5,
    maxX: e.x + 5, maxY: e.y + 5,
  })
  showTooltip(hits[0])
}
```

### 7. ⭐⭐ requestAnimationFrame 节流（必须用）

**RAF 在本场景的两个作用**：

| 用途 | 不用会怎样 |
|------|-----------|
| **渲染节流** | 一帧内多次缩放 → 多次重渲染 → 卡顿 |
| **悬停节流** | mousemove 一秒触发 100+ 次 → 主线程过载 |

```js
// ① 渲染节流：缩放/拖动一帧多次 → 合并成一次渲染
let pending = false
function scheduleRender() {
  if (pending) return
  pending = true
  requestAnimationFrame(() => {
    worker.postMessage({ type: 'render', viewport })
    pending = false
  })
}

// ② 悬停节流：mousemove 合并到每帧一次
let hoverRAF = null
canvas.onmousemove = e => {
  if (hoverRAF) cancelAnimationFrame(hoverRAF)
  hoverRAF = requestAnimationFrame(() => handleHover(e))
}
```

**为啥不用 `setTimeout(fn, 16)`？**
- RAF 跟浏览器刷新率同步（60Hz / 120Hz 自适应）
- 页面切到后台自动暂停（省电）
- 多次 RAF 自动合并，不会堆积

---

### 8. ⭐ 增量更新（实时点云）

LiDAR 实时扫描场景，**不要全量重画**，用环形缓冲：

```js
let writeIdx = 0
const MAX_POINTS = 100000

function appendPoints(newPoints) {
  for (let i = 0; i < newPoints.length; i++) {
    positions[writeIdx * 3]     = newPoints[i].x
    positions[writeIdx * 3 + 1] = newPoints[i].y
    positions[writeIdx * 3 + 2] = newPoints[i].intensity
    writeIdx = (writeIdx + 1) % MAX_POINTS
  }
  scheduleRender()
}
```

### 1. HTML 入口

### 2. 主入口 `main.js`

```js
import { drawGrid } from './bg.js'
import { CloudController } from './cloud-controller.js'
import { setupInteraction } from './interaction.js'

async function main() {
  const bgCanvas    = document.getElementById('bg')
  const cloudCanvas = document.getElementById('cloud')
  const hoverCanvas = document.getElementById('hover')
  const W = 1200, H = 800

  // ① 背景静态层
  drawGrid(bgCanvas.getContext('2d'), W, H)

  // ② 加载二进制点云
  const positions = await loadCloudData('/api/cloud.bin')
  console.log(`加载 ${positions.length / 3} 个点`)

  // ③ 点云渲染（Worker）
  const controller = new CloudController(cloudCanvas, positions)
  await controller.init()

  // ④ 交互层
  setupInteraction(hoverCanvas, positions, controller)
}

async function loadCloudData(url) {
  const res = await fetch(url)
  const buffer = await res.arrayBuffer()
  return new Float32Array(buffer)  // [x, y, intensity, ...]
}

main()
```

### 3. 背景层 `bg.js`

```js
export function drawGrid(ctx, w, h, gridSize = 50) {
  ctx.fillStyle = '#0a0a0a'
  ctx.fillRect(0, 0, w, h)

  ctx.strokeStyle = '#1f1f1f'
  ctx.lineWidth = 1
  ctx.beginPath()
  for (let x = 0; x <= w; x += gridSize) {
    ctx.moveTo(x, 0); ctx.lineTo(x, h)
  }
  for (let y = 0; y <= h; y += gridSize) {
    ctx.moveTo(0, y); ctx.lineTo(w, y)
  }
  ctx.stroke()
}
```

### 4. 主线程控制器 `cloud-controller.js`

```js
export class CloudController {
  constructor(canvas, positions) {
    this.canvas = canvas
    this.positions = positions
    this.worker = null
    this.pending = false
    this.viewport = { x: 0, y: 0, zoom: 1 }
  }

  async init() {
    const offscreen = this.canvas.transferControlToOffscreen()
    this.worker = new Worker(new URL('./cloud-worker.js', import.meta.url), { type: 'module' })

    // 关键：positions.buffer 走 Transferable，零拷贝
    // ⚠️ 如果主线程后续还要用，传过去时用 positions.slice() 复制一份
    this.worker.postMessage(
      {
        type: 'init',
        canvas: offscreen,
        positions: this.positions,
        w: this.canvas.width,
        h: this.canvas.height,
      },
      [offscreen, this.positions.buffer]
    )
  }

  updateViewport(viewport) {
    this.viewport = viewport
    this.scheduleRender()
  }

  scheduleRender() {
    if (this.pending) return
    this.pending = true
    requestAnimationFrame(() => {
      this.worker.postMessage({ type: 'render', viewport: this.viewport })
      this.pending = false
    })
  }
}
```

### 5. Worker 渲染器 `cloud-worker.js`

```js
let canvas, ctx, positions, W, H

self.onmessage = ({ data }) => {
  if (data.type === 'init') {
    canvas = data.canvas
    ctx = canvas.getContext('2d')
    positions = data.positions
    W = data.w
    H = data.h
    render({ x: 0, y: 0, zoom: 1 })
  } else if (data.type === 'render') {
    render(data.viewport)
  }
}

function render(viewport) {
  const { x: vx, y: vy, zoom } = viewport
  const cx = W / 2, cy = H / 2

  // 核心：ImageData 直接写像素
  const imageData = ctx.createImageData(W, H)
  const data = imageData.data

  // LOD 抽样
  const step = zoom < 0.5 ? 10 : zoom < 1 ? 3 : 1

  for (let i = 0; i < positions.length; i += 3 * step) {
    const wx = positions[i]
    const wy = positions[i + 1]
    const intensity = positions[i + 2]

    // 世界坐标 → 屏幕坐标
    const sx = ((wx - vx) * zoom + cx) | 0   // | 0 = 快速取整
    const sy = ((wy - vy) * zoom + cy) | 0

    // 视口剔除
    if (sx < 0 || sx >= W || sy < 0 || sy >= H) continue

    // 按反射强度上色（蓝 → 红渐变）
    const idx = (sy * W + sx) * 4
    data[idx]     = intensity * 255
    data[idx + 1] = (1 - intensity) * 100
    data[idx + 2] = (1 - intensity) * 255
    data[idx + 3] = 255
  }

  ctx.putImageData(imageData, 0, 0)
}
```

### 6. 交互层 `interaction.js`

```js
import RBush from 'rbush'

export function setupInteraction(hoverCanvas, positions, controller) {
  const ctx = hoverCanvas.getContext('2d')
  const W = hoverCanvas.width, H = hoverCanvas.height

  // 空间索引（一次性建立）
  const tree = new RBush()
  const items = []
  for (let i = 0; i < positions.length; i += 3) {
    items.push({
      minX: positions[i], minY: positions[i + 1],
      maxX: positions[i], maxY: positions[i + 1],
      idx: i,
    })
  }
  tree.load(items)

  const viewport = { x: 0, y: 0, zoom: 1 }
  let dragging = false, lastX = 0, lastY = 0
  let hoverRAF = null

  // 拖动
  hoverCanvas.onmousedown = e => {
    dragging = true; lastX = e.offsetX; lastY = e.offsetY
  }
  hoverCanvas.onmouseup = () => dragging = false

  // 移动（拖动 + 悬停查询）
  hoverCanvas.onmousemove = e => {
    if (dragging) {
      viewport.x -= (e.offsetX - lastX) / viewport.zoom
      viewport.y -= (e.offsetY - lastY) / viewport.zoom
      lastX = e.offsetX; lastY = e.offsetY
      controller.updateViewport(viewport)
    } else {
      if (hoverRAF) cancelAnimationFrame(hoverRAF)
      hoverRAF = requestAnimationFrame(() => handleHover(e))
    }
  }

  // 缩放
  hoverCanvas.onwheel = e => {
    e.preventDefault()
    const delta = e.deltaY > 0 ? 0.9 : 1.1
    viewport.zoom *= delta
    controller.updateViewport(viewport)
  }

  function handleHover(e) {
    const wx = (e.offsetX - W / 2) / viewport.zoom + viewport.x
    const wy = (e.offsetY - H / 2) / viewport.zoom + viewport.y
    const r = 3 / viewport.zoom

    const hits = tree.search({
      minX: wx - r, minY: wy - r, maxX: wx + r, maxY: wy + r,
    })

    ctx.clearRect(0, 0, W, H)
    if (hits.length === 0) return

    const hit = hits[0]
    const sx = (positions[hit.idx] - viewport.x) * viewport.zoom + W / 2
    const sy = (positions[hit.idx + 1] - viewport.y) * viewport.zoom + H / 2

    // 高亮圈
    ctx.strokeStyle = '#ffeb3b'
    ctx.lineWidth = 2
    ctx.beginPath()
    ctx.arc(sx, sy, 8, 0, Math.PI * 2)
    ctx.stroke()

    // tooltip
    ctx.fillStyle = '#fff'
    ctx.font = '12px monospace'
    ctx.fillText(
      `x:${positions[hit.idx].toFixed(2)} y:${positions[hit.idx + 1].toFixed(2)}`,
      sx + 15, sy
    )
  }
}
```

## 六、生产坑提醒

| 坑 | 解决 |
|---|------|
| **OffscreenCanvas Safari < 16.4 不支持** | 降级：主线程渲染 + `setTimeout(render, 0)` 分帧 |
| **`transferControlToOffscreen` 只能调一次** | 想销毁重建得换 canvas 元素 |
| **positions.buffer 转移后主线程没了** | 双缓冲：发一份给 Worker，主线程留 `slice()` 副本给 RBush |
| **DPR 高分屏放大模糊** | `canvas.width = w * devicePixelRatio` + `ctx.scale(dpr, dpr)` |
| **内存暴涨** | 不可见区域的点也在内存 → 上分块加载（Tile 方案） |
| **数据更新频繁** | 局部 `putImageData(imageData, dx, dy, dirtyX, dirtyY, dirtyW, dirtyH)` 只刷脏区 |
| **半透明叠加错误** | ImageData 不会自动 alpha blend，要么手算 RGBA，要么用 `globalCompositeOperation` |

## 七、面试回答模板

> 「我会用**主线程 + Worker + 三层 canvas** 的架构：
>
> 1. **后端发二进制 `Float32Array`**（不发 JSON），前端零解析
> 2. **三层 canvas**：背景层（静态网格）/ 点云层（OffscreenCanvas 给 Worker）/ 交互层（悬停高亮）
> 3. **核心渲染走 Web Worker + OffscreenCanvas**，主线程不卡顿；用 `Transferable Objects` 零拷贝传 `ArrayBuffer`
> 4. **像素级写入**：`createImageData` + `putImageData` 直接操作底层像素数组，比 `fillRect` 快 10-50 倍
> 5. **视口剔除 + LOD 抽样**：屏幕外的不画，缩得小时按 `1/3` 或 `1/10` 抽样
> 6. **交互查询用 RBush 四叉树**：O(log n) 悬停命中，60fps 流畅
> 7. **`requestAnimationFrame` 节流**：多次缩放/拖动合并成一帧
>
> 10w 点首屏 < 10ms，交互 60fps 稳定。要更多就上分块加载（Tile）+ 不可见区域 GC。
>
> 一句话：**canvas 2D 画 10w 点没问题，关键是别用 `arc/fillRect` 一个个画，直接写底层像素数组 + 丢 Worker 不阻塞主线程。**」

## 八、高频追问

1. **「为啥不用 WebGL？」**
   → 历史选型 / 团队栈 / 简单场景 canvas 够用。但要表明**知道 WebGL（Three.js / deck.gl / Potree）是更优解**，能撑百万级到亿级。

2. **「ImageData 怎么处理半透明叠加？」**
   → 自己实现 alpha blend（`new = old * (1-a) + src * a`），或者用临时 canvas + `globalCompositeOperation`。

3. **「点很大（3x3 像素）怎么办？」**
   → 一次写 9 个邻居像素 / 预生成"点 sprite" 用 `drawImage` 批量贴图。

4. **「热力图/密度图？」**
   → 累加到一个 `Float32Array` 计数器，最后映射 colormap 写入 ImageData。

5. **「移动端性能？」**
   → DPR 降到 1（不用 retina）、降采样、关交互动画、用 `OffscreenCanvas` 在 Worker 渲染。

6. **「数据是流式来的怎么办？」**
   → 环形缓冲 + 局部 `putImageData` 脏矩形刷新，或者新数据攒一批后整体重绘。

7. **「`requestAnimationFrame` 一定要用吗？」**
   → 静态点云只在交互时节流用；实时点云要全程同步刷新率；纯一次性显示可不用。
