# 进程和线程的区别？
- 进程
    进程是对正在运行的程序的一个抽象，是系统进行资源分配和调度的基本单位。
    - 程序用于描述进程要完成的功能，是控制进程执行的指令集。
    - 数据集合是在执行时所需要的数据和工作区
    - 程序控制块，包含进程的描述信息和控制信息，是进程存在的唯一标志。
- 线程
    - 线程是操作系统进行调度的最小单位，是进程中的一个执行单位，负责当前进程中的执行。
## 区别
本质区别：
    - 进程是操作系统资源分配的基本单位，而线程是任务调度和执行的基本单位。
## Q: 什么是死锁？死锁的四个必要条件是什么？如何预防？

**A:**

**死锁**是指多个进程（或线程）因互相占有对方所需要的资源，同时都在等待对方释放，导致全部陷入永久阻塞的状态。

**死锁的四个必要条件（缺一不可）：**

| 条件 | 说明 |
|------|------|
| **互斥条件（Mutual Exclusion）** | 资源一次只能被一个进程使用 |
| **请求与保持（Hold and Wait）** | 进程持有已有资源，同时请求其他资源，等待期间不释放 |
| **不可抢占（No Preemption）** | 资源只能由持有者主动释放，不能被强制抢占 |
| **循环等待（Circular Wait）** | 进程之间形成环形依赖链，每个进程都等待下一个进程持有的资源 |

**预防死锁的方法（破坏至少一个必要条件）：**

1. **破坏请求与保持**：进程一次性申请所有所需资源（全部成功才执行），缺点：资源利用率低
2. **破坏不可抢占**：若进程申请新资源失败，必须释放已有资源再重新申请
3. **破坏循环等待（最常用）**：对所有资源编号，进程按编号从小到大顺序申请，不允许反向申请

**检测与恢复：**

- **检测**：资源分配图中如果存在环，说明可能死锁
- **恢复**：选择代价最小的进程回滚（银行家算法）

**前端中的死锁类比：**

```js
// 两个 Promise 相互等待 → 逻辑死锁
let resolveA, resolveB

const promiseA = new Promise(resolve => { resolveA = resolve })
const promiseB = new Promise(resolve => { resolveB = resolve })

// A 等 B，B 等 A
promiseA.then(() => resolveB())
promiseB.then(() => resolveA())

// 没有人先触发，两者永远等待
```
