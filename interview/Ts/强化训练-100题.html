TypeScript 类型体操 100 题（含答案） ​

由浅入深的 TS 类型训练题集，按 7 个阶段递进：基础类型 → 函数泛型 → keyof/typeof → 工具类型手写 → 条件类型与 infer → 映射类型与模板字面量 → 综合挑战。

每题给出题目与参考答案。建议先自己写，再对答案。第 04 / 05 / 07 阶段是面试手写题重灾区，务必反复练。

🤖 AI 陪练提示词（复制到 AI 对话即可开练） ​

【出题规则】
1. 一次只出一道题，从 Q1 开始，按题号顺序推进，不要跳题、不要剧透后面的题。
2. 每道题只给我看：题号 + 题目说明 + 输入/输出示例。先不要给答案。

【批改规则】

第 01 阶段 · 基础类型（Q1-Q15） ​

目标：字面量、联合、元组、可选、只读、字面量推断。

Q1. 只接受 'red' | 'green' | 'blue' 的类型 ​

题目：定义一个类型，只允许取 'red'、'green'、'blue' 这三个字符串字面量之一。

示例：

  • const c: Q1 = 'red' ✅
  • const c: Q1 = 'yellow' ❌

答案：

ts
type Q1 = 'red' | 'green' | 'blue'

Q2. 元组类型 [string, number, boolean] ​

题目：定义一个长度固定、每个位置类型也固定的元组。

示例：

  • const t: Q2 = ['a', 1, true] ✅
  • const t: Q2 = ['a', 1] ❌（缺第三个元素）

答案：

ts
type Q2 = [string, number, boolean]

Q3. 对象类型：name 必填 string，age 可选 number ​

题目：定义对象类型，name 必填，age 可以不写。

示例：

  • const u: Q3 = { name: 'a' } ✅
  • const u: Q3 = { name: 'a', age: 1 } ✅
  • const u: Q3 = {} ❌（缺 name）

答案：

ts
type Q3 = { name: string; age?: number }

Q4. 把所有字段改成 readonly ​

题目：定义对象类型，id 和 title 都不可修改。

示例：

  • const q: Q4 = { id: 1, title: 'a' } ✅
  • q.id = 2 ❌（只读，不能赋值）

答案：

ts
type Q4 = { readonly id: number; readonly title: string }

Q5. 要么 string，要么 string[] ​

题目：定义一个类型，值可以是单个字符串或字符串数组。

示例：

  • const v: Q5 = 'a' ✅
  • const v: Q5 = ['a', 'b'] ✅
  • const v: Q5 = 1 ❌

答案：

ts
type Q5 = string | string[]

Q6. 推导 { a: 1, b: 'hi' } as const 的类型 ​

题目：用 typeof 取出加了 as const 的对象的类型（字面量 + 只读）。

示例：

  • 输入：{ a: 1, b: 'hi' } as const
  • 输出：{ readonly a: 1; readonly b: 'hi' }

答案：

ts
const obj = { a: 1, b: 'hi' } as const
type Q6 = typeof obj // { readonly a: 1; readonly b: 'hi' }

Q7. 联合类型去除 null / undefined ​

题目：从联合类型中剔除 null 和 undefined。

示例：

  • 输入：string | number | null | undefined
  • 输出：string | number

答案：

ts
type Input = string | number | null | undefined
type Q7 = NonNullable<Input>            // string | number
// 等价：Exclude<Input, null | undefined>

Q8. 元组转联合类型 ​

题目：把元组的所有元素类型变成联合类型。

示例：

  • 输入：['a', 'b', 'c']
  • 输出：'a' | 'b' | 'c'

答案：

ts
type Q8 = ['a', 'b', 'c'][number]        // 'a' | 'b' | 'c'

Q9. 元组长度 ​

题目：取出元组的长度（字面量数字）。

示例：

  • 输入：[string, number, boolean]
  • 输出：3

答案：

ts
type Q9 = [string, number, boolean]['length']  // 3

Q10. 取元组首元素 ​

题目：实现 First<T>，取出元组第一个元素类型；空元组返回 never。

示例：

  • First<[3, 2, 1]> → 3
  • First<[]> → never

答案：

ts
type First<T extends readonly any[]> =
  T extends readonly [infer F, ...any[]] ? F : never

Q11. 取元组末元素 ​

题目：实现 Last<T>，取出元组最后一个元素类型。

示例：

  • Last<[3, 2, 1]> → 1
  • Last<[]> → never

答案：

ts
type Last<T extends readonly any[]> =
  T extends readonly [...any[], infer L] ? L : never

Q12. 数组类型 → 元素类型 ​

题目：实现 ElementType<T>，从数组类型提取元素类型。

示例：

  • ElementType<string[]> → string
  • ElementType<number[]> → number

答案：

ts
type ElementType<T> = T extends (infer U)[] ? U : never

Q13. 判断类型是否为 string ​

题目：实现 IsString<T>，是 string 返回 true，否则 false。

示例：

  • IsString<string> → true
  • IsString<number> → false

答案：

ts
type IsString<T> = T extends string ? true : false

Q14. If<C, T, F>：C 为 true 返回 T，否则 F ​

题目：实现类型版三元表达式，C 为 true 取 T，否则取 F。

示例：

  • If<true, 'a', 'b'> → 'a'
  • If<false, 'a', 'b'> → 'b'

答案：

ts
type If<C extends boolean, T, F> = C extends true ? T : F

Q15. Concat：拼接两个元组 ​

题目：实现 Concat<A, B>，拼接两个元组。

示例：

  • Concat<[1, 2], [3, 4]> → [1, 2, 3, 4]

答案：

ts
type Concat<A extends readonly any[], B extends readonly any[]> = [...A, ...B]

第 02 阶段 · 函数与泛型（Q16-Q30） ​

目标：泛型约束、默认值、参数推断、重载、函数类型操作。

Q16. 泛型 identity 函数类型 ​

题目：定义泛型函数类型，传入什么类型就返回什么类型。

示例：

  • const fn: Identity = x => x
  • fn(1) → number；fn('a') → string

答案：

ts
type Identity = <T>(arg: T) => T

Q17. 给泛型加约束：T 必须有 length ​

题目：函数泛型参数 T 被约束为「必须有 length: number 属性」。

示例：

  • logLen('abc') ✅；logLen([1, 2]) ✅
  • logLen(123) ❌（number 没有 length）

答案：

ts
function logLen<T extends { length: number }>(arg: T): void {}

Q18. 给泛型 T 加默认值 string ​

题目：定义 Box<T>，不传泛型参数时 T 默认为 string。

示例：

  • Box → { value: string }
  • Box<number> → { value: number }

答案：

ts
type Box<T = string> = { value: T }

Q19. 提取函数参数类型为元组（手写 Parameters） ​

题目：手写 MyParameters<T>，把函数的参数列表提取为元组。

示例：

  • MyParameters<(a: string, b: number) => void> → [string, number]

答案：

ts
type MyParameters<T> = T extends (...args: infer P) => any ? P : never

Q20. 提取函数返回值类型（手写 ReturnType） ​

题目：手写 MyReturnType<T>，提取函数的返回值类型。

示例：

  • MyReturnType<() => string> → string

答案：

ts
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never

Q21. 递归解包 Promise（手写 Awaited） ​

题目：手写 MyAwaited<T>，递归剔掉嵌套 Promise，取出最终 resolve 类型。

示例：

  • MyAwaited<Promise<Promise<number>>> → number

答案：

ts
type MyAwaited<T> = T extends Promise<infer U> ? MyAwaited<U> : T

Q22. 提取函数的 this 类型 ​

题目：提取函数声明中 this 参数的类型。

示例：

  • ThisParameterType<(this: Window, n: number) => void> → Window

答案：

ts
type ThisParameterType<T> =
  T extends (this: infer U, ...args: any[]) => any ? U : never

Q23. 函数重载提取最后一个签名的返回值 ​

题目：对于有多个重载签名的函数，ReturnType 默认推断哪个签名的返回值？

示例：

  • 重载 f(): string / f(x: number): number
  • ReturnType<typeof f> → 取最后一个签名的返回值

答案：

ts
type Q23 = ReturnType<typeof f_q23> // TS 重载推断默认取最后一个

Q24. Curry 化（仅 2 参数） ​

题目：把二元函数类型柯里化为 (a) => (b) => R。

示例：

  • Curry2<(a: string, b: number) => boolean> → (a: string) => (b: number) => boolean

答案：

ts
type Curry2<F> = F extends (a: infer A, b: infer B) => infer R
  ? (a: A) => (b: B) => R
  : never

Q25. PromiseAll 的类型 ​

题目：为 promiseAll 函数定义类型，输入一组（可能是 Promise 的）值，返回 Promise<解包后的元组>。

示例：

  • promiseAll([1, Promise.resolve('a'), true]) → Promise<[number, string, boolean]>

答案：

ts
declare function promiseAll<T extends readonly any[]>(
  values: readonly [...T]
): Promise<{ [K in keyof T]: T[K] extends Promise<infer U> ? U : T[K] }>

Q26. 函数组合 Compose（2 个函数）f(g(x)) ​

题目：组合两个函数，g 的返回值作为 f 的输入，结果是 (a) => f(g(a))。

示例：

  • Compose2<(b: number) => boolean, (a: string) => number> → (a: string) => boolean

答案：

ts
type Compose2<F, G> = F extends (b: infer B) => infer C
  ? G extends (a: infer A) => B
    ? (a: A) => C
    : never
  : never

Q27. 元组转函数参数 ​

题目：把元组类型展开作为函数的参数列表。

示例：

  • TupleToFn<[string, number]> → (args_0: string, args_1: number) => void

答案：

ts
type TupleToFn<T extends readonly any[]> = (...args: [...T]) => void

Q28. 提取对象中所有方法名 ​

题目：提取对象中值为函数的那些 key（联合）。

示例：

  • MethodKeys<{ a: number; fn(): void; go(): void }> → 'fn' | 'go'

答案：

ts
type MethodKeys<T> = {
  [K in keyof T]: T[K] extends (...args: any) => any ? K : never
}[keyof T]

Q29. 手写 Capitalize（首字母大写） ​

题目：手写 MyCapitalize<S>，把字符串首字母变大写。

示例：

  • MyCapitalize<'hello'> → 'Hello'

答案：

ts
type MyCapitalize<S extends string> =
  S extends `${infer First}${infer Rest}` ? `${Uppercase<First>}${Rest}` : S

Q30. 函数参数前追加一个 user 参数 ​

题目：实现 AppendArgument<F, A>，在原函数参数最前面插入一个 user: A 参数。

示例：

  • AppendArgument<(n: number) => void, string> → (user: string, n: number) => void

答案：

ts
type AppendArgument<F, A> = F extends (...args: infer P) => infer R
  ? (user: A, ...args: P) => R
  : never

第 03 阶段 · keyof / typeof / 索引访问（Q31-Q45） ​

目标：在对象/数组/常量之间互相提取与组合类型。

Q31. 提取对象所有 key（联合） ​

题目：用 keyof 取出对象类型所有键的联合。

示例：

  • User = { id; name; age; email }
  • 输出：'id' | 'name' | 'age' | 'email'

答案：

ts
type Q31 = keyof User // 'id' | 'name' | 'age' | 'email'

Q32. 提取对象所有 value 类型（联合） ​

题目：用 T[keyof T] 取出对象所有属性值类型的联合。

示例：

  • User = { id: number; name: string }
  • 输出：number | string

答案：

ts
type Q32 = User[keyof User]

Q33. 索引访问取嵌套属性 ​

题目：用索引访问逐层取出嵌套属性的类型。

示例：

  • { a: string; b: { c: number } }['b']['c'] → number

答案：

ts
type Q33 = { a: string; b: { c: number } }['b']['c'] // number

Q34. 数组 → 元素联合类型 ​

题目：用 typeof arr[number] 把数组值转为元素联合类型。

示例：

  • const arr = ['a', 'b'] as const
  • 输出：'a' | 'b'

答案：

ts
type Q34 = typeof arr[number]

Q35. typeof 从值倒推类型 ​

题目：用 typeof 从一个变量值推导出它的类型。

示例：

  • const config = { port: 3000, host: 'x' }
  • 输出：{ port: number; host: string }

答案：

ts
type Q35 = typeof config // { port: number; host: string }

Q36. 字面量精确化 ​

题目：配合 as const，用 typeof 取出字面量类型而非宽类型。

示例：

  • const shape = { kind: 'circle' } as const
  • typeof shape['kind'] → 'circle'（而非 string）

答案：

ts
type Q36 = typeof shape['kind'] // 'circle'（而非 string）

Q37. 手写 Pick ​

题目：手写 MyPick<T, K>，从 T 中选取指定的几个属性 K。

示例：

  • MyPick<{ a: number; b: string; c: boolean }, 'a' | 'b'> → { a: number; b: string }

答案：

ts
type MyPick<T, K extends keyof T> = { [P in K]: T[P] }

Q38. 手写 Omit ​

题目：手写 MyOmit<T, K>，从 T 中排除指定的几个属性 K。

示例：

  • MyOmit<{ a: number; b: string; c: boolean }, 'a'> → { b: string; c: boolean }

答案：

ts
type MyOmit<T, K extends keyof any> = {
  [P in keyof T as P extends K ? never : P]: T[P]
}

Q39. 提取值类型为 string 的 key ​

题目：提取对象中值类型为 string 的那些 key（联合）。

示例：

  • StringKeys<{ a: string; b: number; c: string }> → 'a' | 'c'

答案：

ts
type StringKeys<T> = {
  [K in keyof T]: T[K] extends string ? K : never
}[keyof T]

Q40. 仅保留值类型为 string 的属性 ​

题目：用 as 重映射过滤，只保留值类型为 string 的属性。

示例：

  • FilterStringProps<{ a: string; b: number; c: string }> → { a: string; c: string }

答案：

ts
type FilterStringProps<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K]
}

Q41. key 改为大写 ​

题目：用 as 重映射 + Uppercase，把所有 key 变大写。

示例：

  • UppercaseKeys<{ name: string; age: number }> → { NAME: string; AGE: number }

答案：

ts
type UppercaseKeys<T> = {
  [K in keyof T as Uppercase<K & string>]: T[K]
}

Q42. 交换 key 和 value ​

题目：翻转对象，把 key 和 value 互换（value 需是字符串）。

示例：

  • Flip<{ a: 'x'; b: 'y' }> → { x: 'a'; y: 'b' }

答案：

ts
type Flip<T extends Record<string, string>> = {
  [K in keyof T as T[K]]: K
}

Q43. 所有 key 加前缀 on 并首字母大写 ​

题目：用 as 重映射 + 模板字面量，给每个 key 加 on 前缀并首字母大写。

示例：

  • Eventify<{ click: 1; hover: 2 }> → { onClick: 1; onHover: 2 }

答案：

ts
type Eventify<T> = {
  [K in keyof T as `on${Capitalize<K & string>}`]: T[K]
}

Q44. 手写 InstanceType ​

题目：手写 MyInstanceType<T>，提取构造函数的实例类型。

示例：

  • class C {}；MyInstanceType<typeof C> → C

答案：

ts
type MyInstanceType<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: any) => infer R ? R : never

Q45. 手写 ConstructorParameters ​

题目：手写 MyConstructorParameters<T>，提取构造函数的参数元组。

示例：

  • class C { constructor(a: string, b: number) {} }
  • MyConstructorParameters<typeof C> → [string, number]

答案：

ts
type MyConstructorParameters<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: infer P) => any ? P : never

第 04 阶段 · 内置工具类型手写实现（Q46-Q65） ​

目标：能闭着眼睛默写 TS 全部内置工具类型 —— 面试手写题重灾区！

Q46. MyPartial：所有属性可选 ​

题目：手写 MyPartial<T>，把所有属性变为可选。

示例：

  • MyPartial<{ a: number; b: string }> → { a?: number; b?: string }

答案：

ts
type MyPartial<T> = { [K in keyof T]?: T[K] }

Q47. MyRequired：所有属性必填 ​

题目：手写 MyRequired<T>，用 -? 去掉可选修饰。

示例：

  • MyRequired<{ a?: number; b?: string }> → { a: number; b: string }

答案：

ts
type MyRequired<T> = { [K in keyof T]-?: T[K] }

Q48. MyReadonly：所有属性只读 ​

题目：手写 MyReadonly<T>，把所有属性变只读。

示例：

  • MyReadonly<{ a: number }> → { readonly a: number }

答案：

ts
type MyReadonly<T> = { readonly [K in keyof T]: T[K] }

Q49. Mutable：去掉所有 readonly ​

题目：手写 Mutable<T>，用 -readonly 去掉只读修饰。

示例：

  • Mutable<{ readonly a: number }> → { a: number }

答案：

ts
type Mutable<T> = { -readonly [K in keyof T]: T[K] }

Q50. MyRecord ​

题目：手写 MyRecord<K, V>，构造键为 K、值为 V 的对象类型。

示例：

  • MyRecord<'a' | 'b', number> → { a: number; b: number }

答案：

ts

Q74. UnionToIntersection：联合 → 交叉 ​

题目：实现 UnionToIntersection<U>，把联合类型转为交叉类型。

ts

  (U extends any ? (x: U) => void : never) extends (x: infer I) => void ? I : never

核心技巧：函数参数逆变。把联合摊进函数参数位，infer 回来时被合并成交叉。

Q75. IsNever（用 [T] 禁分发） ​

答案：

ts
type MyIsNever<T> = [T] extends [never] ? true : false

Q76. IsAny ​

type MyIsAny<T> = 0 extends 1 & T ? true : false

只有 T = any 时 1 & T 才会塌缩成 any，使 0 extends any 成立。

Q78. Includes：元组是否包含 U（严格相等） ​

题目：实现 Includes<T, U>，判断元组是否包含某类型（需严格相等）。

示例：

  • Includes<[1, 2, 3], 2> → true；Includes<[1, 2], 3> → false

答案：

ts
type Equal<X, Y> =
  (<T>() => T extends X ? 1 : 2) extends (<T>() => T extends Y ? 1 : 2) ? true : false

type Includes<T extends readonly any[], U> =
  T extends readonly [infer F, ...infer R]
    ? Equal<F, U> extends true ? true : Includes<R, U>
    : false

答案：

ts
type Flatten<T extends readonly any[]> =
  T extends readonly [infer F, ...infer R]
    ? F extends readonly any[]
      ? [...F, ...Flatten<R>]
      : [F, ...Flatten<R>]
    : []

Q86. Replace：替换第一次出现 ​

题目：实现 Replace<S, From, To>，把字符串 S 中第一个 From 替换为 To。

答案：

ts
type Replace<S extends string, From extends string, To extends string> =
  From extends ''
    ? S
    : S extends `${infer L}${From}${infer R}`
      ? `${L}${To}${R}`
      : S

Q87. ReplaceAll：替换所有出现（递归剩余部分） ​

答案：

ts
type ReplaceAll<S extends string, From extends string, To extends string> =
  From extends ''
    ? S
    : S extends `${infer L}${From}${infer R}`
      ? `${L}${To}${ReplaceAll<R, From, To>}`
      : S

Q88. Trim：删除两侧空白 ​

题目：实现 Trim<S>，递归删除字符串两侧的空格/换行/制表符。

示例：

  • Trim<' hi '> → 'hi'

答案：

ts
type WS = ' ' | '\n' | '\t'
type TrimLeft<S extends string> = S extends `${WS}${infer R}` ? TrimLeft<R> : S
type TrimRight<S extends string> = S extends `${infer L}${WS}` ? TrimRight<L> : S
type Trim<S extends string> = TrimRight<TrimLeft<S>>

Q89. Split：按分隔符切分成元组 ​

题目：实现 Split<S, Sep>，类似 JS 的 split，按分隔符切成字符串元组。

示例：

  • Split<'a,b,c', ','> → ['a', 'b', 'c']

答案：

ts
type Split<S extends string, Sep extends string> =
  Sep extends ''
    ? S extends `${infer A}${infer B}`
      ? B extends '' ? [A] : [A, ...Split<B, ''>]
      : []
    : S extends `${infer A}${Sep}${infer B}`
      ? [A, ...Split<B, Sep>]
      : [S]

Q90. Join：元组用 Sep 拼接成字符串 ​

题目：实现 Join<T, Sep>，类似 JS 的 join，把字符串元组用分隔符拼接。

示例：

  • Join<['a', 'b', 'c'], '-'> → 'a-b-c'

答案：

ts
type Join<T extends readonly string[], Sep extends string> =
  T extends [infer F extends string, ...infer R extends string[]]
    ? R extends [] ? F : `${F}${Sep}${Join<R, Sep>}`
    : ''

Q91. CamelCase：连字符 → 驼峰 ​

题目：实现 CamelCase<S>，把连字符命名转为驼峰命名。

示例：

  • CamelCase<'foo-bar-baz'> → 'fooBarBaz'

答案：

ts
type CamelCase<S extends string> =
  S extends `${infer A}-${infer B}${infer C}`
    ? `${A}${Uppercase<B>}${CamelCase<C>}`
    : S

Q92. KebabCase：驼峰 → 连字符 ​

题目：实现 KebabCase<S>，把驼峰命名转为连字符命名。

示例：

  • KebabCase<'fooBarBaz'> → 'foo-bar-baz'

答案：

ts
type KebabCase<S extends string> =
  S extends `${infer A}${infer B}`
    ? B extends Uncapitalize<B>
      ? `${Uncapitalize<A>}${KebabCase<B>}`
      : `${Uncapitalize<A>}-${KebabCase<B>}`
    : S

第 07 阶段 · 综合类型体操（Q93-Q100） ​

目标：每题综合前面所有知识 —— 决战面试场。

Q93. Chainable Options（链式累积类型） ​

题目：实现 Chainable，支持链式 .option(key, value) 不断累积键，最后 .get() 返回完整对象类型。

示例：

  • chain.option('a', 1).option('b', 'x').get() → { a: number; b: string }

答案：

ts
type Chainable<O = {}> = {
  option<K extends string, V>(
    key: K extends keyof O ? never : K,
    value: V
  ): Chainable<O & { [P in K]: V }>
  get(): { [K in keyof O]: O[K] }
}

精髓：方法返回携带更新泛型的自己 Chainable<O & 新键>，实现链式累积。

Q94. DeepGet：按路径深度取值 ​

题目：实现 DeepGet<T, Path>，按 'a.b.c' 的点路径逐层取出嵌套属性类型。

示例：

  • DeepGet<{ a: { b: { c: number } } }, 'a.b.c'> → number

答案：

ts
type DeepGet<T, Path extends string> =
  Path extends `${infer K}.${infer Rest}`
    ? K extends keyof T
      ? DeepGet<T[K], Rest>
      : never
    : Path extends keyof T
      ? T[Path]
      : never

深度 = 递归换 T，每层 keyof 只看当前层。

Q95. ParseURLParams：提取所有 :param ​

题目：实现 ParseURLParams<S>，从路由路径中提取所有 :param 参数名（联合）。

示例：

  • ParseURLParams<'/user/:id/post/:postId'> → 'id' | 'postId'

答案：

ts
type ParseURLParams<S extends string> =
  S extends `${string}:${infer P}/${infer Rest}`
    ? P | ParseURLParams<`/${Rest}`>
    : S extends `${string}:${infer P}`
      ? P
      : never

Q96. EventBus：emit/on 类型安全 ​

题目：定义类型安全的 EventBus<E>，on/emit 的事件名和参数类型联动。

示例：

  • E = { login: [string]; logout: [] }
  • bus.emit('login', 'tom') ✅；bus.emit('login', 1) ❌

答案：

ts
type EventBus<E extends Record<string, any[]>> = {
  on<K extends keyof E>(event: K, handler: (...args: E[K]) => void): void
  emit<K extends keyof E>(event: K, ...args: E[K]): void
}

精髓：...args: E[K] —— 用元组类型当剩余参数。

Q97. CamelCaseKeys：对象 key 全部转驼峰 ​

题目：实现 CamelCaseKeys<T>，把对象的所有 key 转为驼峰命名。

示例：

  • CamelCaseKeys<{ 'user-name': string }> → { userName: string }

答案：

ts
type CamelCaseKeys<T> = {
  [K in keyof T as K extends string ? CamelCase<K> : K]: T[K]
}

Q98. Add：类型层加法（元组长度模拟） ​

题目：实现 Add<A, B>，用元组长度模拟类型层加法。

示例：

  • Add<3, 5> → 8

答案：

ts
type BuildTuple<L extends number, R extends any[] = []> =
  R['length'] extends L ? R : BuildTuple<L, [any, ...R]>

type Add<A extends number, B extends number> =
  [...BuildTuple<A>, ...BuildTuple<B>]['length']

类型层不能算 3 + 5，但能"摆 3 个石子 + 摆 5 个石子，再数总数"。

Q99. ObjectEntries：仿 Object.entries ​

题目：实现 ObjectEntries<T>，返回 [key, value] 元组的联合（类型版 Object.entries）。

示例：

  • ObjectEntries<{ a: number; b: string }> → ['a', number] | ['b', string]

答案：

ts
type ObjectEntries<T> = { [K in keyof T]-?: [K, T[K]] }[keyof T]

套路：{ [K in keyof T]: X }[keyof T] 是"对象 → 联合"的万能桥。

Q100. JSONValue：递归定义合法 JSON 类型 ​

题目：递归定义 JSONValue，描述任意合法的 JSON 值（可任意嵌套）。

示例：

  • { a: [1, 'x', { b: true, c: null }] } ✅
  • { fn: () => {} } ❌（函数不是合法 JSON）

答案：

ts
type JSONValue =
  | string
  | number
  | boolean
  | null
  | JSONValue[]
  | { [key: string]: JSONValue }

递归自引用：数组元素和对象值都指回 JSONValue，可嵌套任意深度。

贯穿全程的 5 大通用套路 ​

  1. 递归拆解：字符串拆 ${L}x${R}、元组拆 [F, ...R]、对象拆 keyof + T[K]
  2. 左短右贪：模板 infer 最短优先，配合递归处理剩余
  3. 对象 → 联合桥：{ [K in keyof T]: X }[keyof T]
  4. 元组当数字：长度 ['length'] 做算术
  5. 泛型累积：返回"携带更新泛型的自己"实现链式
