Vue 技术要点 ​

基础认知篇 ​

Q: Vue 框架理解 ​

A:

Vue 是一套用于构建用户界面的渐进式框架，核心思想是“数据驱动视图 + 组件化开发”。

  1. 响应式：数据变化后，视图会自动更新。
  2. 组件化：将页面拆分为高复用、低耦合的组件。
  3. 渐进式：可从模板增强逐步过渡到工程化全家桶。
  4. 生态完善：路由、状态管理、构建工具与社区方案成熟。

Q: SPA 单页应用优缺点和实现方式是什么？ ​

A:

SPA（Single Page Application）是单页面应用，首次加载后通过前端路由切换视图，通常不整页刷新。

优点：

  1. 交互流畅，页面切换体验好。
  2. 前后端职责清晰，便于分工。
  3. 更适合组件化和状态管理。

缺点：

  1. 首屏资源可能较大，需要优化加载策略。
  2. SEO 天然较弱，通常要配合 SSR/预渲染。
  3. 路由配置或服务端回退配置不当时容易出现刷新 404。

实现关键：

  1. 路由模式：hash 或 history。
  2. 路由监听：路径变化后渲染对应组件。
  3. 服务端配置：history 模式需将未知路径回退到 index.html。

Q: v-if 和 v-show 的区别是什么？ ​

A:

v-if 是“真正的条件渲染”，v-show 是“显示状态切换”。

  1. v-if：条件不满足时不渲染节点，涉及创建/销毁。
  2. v-show：始终渲染节点，仅切换 display: none。
  3. 高频切换用 v-show 更合适；初次是否展示不确定且切换少，优先 v-if。

Q: v-if 和 v-for 为什么不建议写在同一个元素上？ ​

A:

同时使用会增加模板理解和渲染成本，也容易产生优先级误解（不同版本行为有差异）。

推荐做法：

  1. 先在计算属性中过滤列表，再 v-for。
  2. 或将 v-if 提升到外层容器。

响应式与渲染篇 ​

Q: Vue2 响应式原理 ​

A:

Vue2 基于 Object.defineProperty 劫持对象属性的 get/set，通过依赖收集和派发更新驱动渲染。

  1. getter 触发依赖收集（记录当前 watcher）。
  2. setter 触发通知更新（让 watcher 重新执行）。
  3. 组件更新时会重新执行渲染函数，经过 diff 后只更新必要 DOM。

⚠️ 注意：Vue2 对对象新增属性和数组下标直接赋值的监听能力有限。

Q: 动态属性响应式怎么解决？ ​

A:

Vue2 在初始化阶段才会把属性转成响应式，新加属性默认不会被劫持。

解决方式：

  1. 使用 this.$set(target, key, value)。
  2. 用新对象替换旧对象：this.obj = { ...this.obj, newKey: value }。

Q: nextTick 作用 ​

A:

Vue 会异步批量更新 DOM。nextTick 用来在“本轮 DOM 更新完成后”执行逻辑。

常见场景：

  1. 更新状态后立即读取最新 DOM。
  2. 列表变更后计算高度或滚动位置。
  3. 弹窗展示后自动聚焦输入框。
js
this.visible = true
this.$nextTick(() => {
  this.$refs.input?.focus()
})

Q: 虚拟 DOM 与 diffdiff 和 key 有什么关系？ ​

A:

虚拟 DOM 是用 JavaScript 对象描述 UI 结构。更新时比较新旧虚拟 DOM，最小化真实 DOM 操作。

key 是节点稳定标识，作用是让 diff 更准确地复用与移动节点。

  1. key 稳定且唯一：复用正确，性能更好。
  2. 用索引做 key：列表插入/删除时容易错位复用。

Q: Vue 从 template 到真实 DOM 发生了什么？ ​

A:

核心流程：

  1. template 解析成 AST。
  2. AST 生成 render 函数。
  3. render 生成虚拟 DOM（VNode）。
  4. patch 把 VNode 挂载/更新到真实 DOM。

Q: computed 和 watch 的区别是什么？ ​

A:

  1. computed：用于“派生状态”，有缓存，依赖不变则复用结果。
  2. watch：用于“副作用”，如请求接口、写缓存、手动操作 DOM。
  3. computed 需要返回值；watch 以回调处理变化。

生命周期与组件篇 ​

Q: Vue2 生命周期如何理解？ ​

A:

可以理解为：创建 -> 挂载 -> 更新 -> 销毁。

  1. 创建：beforeCreate、created。
  2. 挂载：beforeMount、mounted。
  3. 更新：beforeUpdate、updated。
  4. 销毁：beforeDestroy、destroyed。
  5. 缓存组件：activated、deactivated。

父子渲染顺序（常见）：

父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted

Q: data 函数式 ​

A:

为了保证每个组件实例拥有独立数据副本，避免多个实例共享同一对象导致相互污染。

Q: keep-alive 缓存有哪些常用配置？ ​

A:

keep-alive 是内置抽象组件，用于缓存动态组件实例，避免重复销毁和重建。

常用配置：

  1. include：仅缓存匹配组件。
  2. exclude：排除匹配组件。
  3. max：最大缓存数量（LRU 策略）。

Q: slot 插槽用法适用场景是什么？ ​

A:

slot 是父组件向子组件分发内容的机制。

  1. 默认插槽。
  2. 具名插槽。
  3. 作用域插槽（子传父数据）。

适用场景：弹窗、表格、列表、表单等高复用且布局可定制的组件。

Q: mixin 的作用和问题有哪些？ ​

A:

mixin 用于复用组件选项（data/methods/lifecycle）。

优点：复用快、改造旧项目成本低。
缺点：命名冲突、来源不透明、维护难度高。

现代项目更推荐 Composition API 的组合函数（useXxx）来替代大量 mixin。

Q: Vue 组件通信方式有哪些？ ​

A:

  1. 父子：props / emit。
  2. 跨层：provide / inject。
  3. 全局共享：Vuex/Pinia。
  4. 少量松耦合事件：事件总线（谨慎使用，注意可维护性）。

Q: 组件 vs 插件 ​

A:

  1. 组件：封装 UI 与交互逻辑，面向页面复用。
  2. 插件：扩展 Vue 全局能力，如全局方法、指令、路由、状态管理。

典型插件形态是 install(app)，通过 app.use() 安装。

工程实战篇 ​

Q: 自定义指令适合解决什么问题？ ​

A:

适合封装“底层 DOM 行为”而不是业务状态。

常见场景：自动聚焦、懒加载、权限控制、拖拽、点击外部关闭等。

js
app.directive('focus', {
  mounted(el) {
    el.focus()
  }
})

Q: Vue2 过滤器（filter）了解吗？ ​

A:

Vue2 中可用于展示层轻量格式化，但 Vue3 已移除过滤器语法。建议改用：

  1. 计算属性。
  2. 方法。
  3. 可复用的格式化函数。

Q: axios 封装通常封装哪些内容？ ​

A:

封装 axios 的目标是统一请求规范和异常处理。

常见封装点：

  1. 基础配置：baseURL、超时、通用头。
  2. 请求拦截：鉴权 token、幂等标识。
  3. 响应拦截：统一业务码判断、错误提示、重试策略。
  4. 能力增强：取消重复请求、上传下载、按模块拆分 API。

Q: 权限管理方案按钮级权限怎么做？ ​

A:

  1. 路由权限：基于角色/菜单动态生成可访问路由。
  2. 页面权限：无权限时重定向或展示 403。
  3. 按钮级权限：
    1. 指令方案：v-permission="'user:create'"。
    2. 组合函数方案：hasPermission('user:create')。
  4. 权限源应由后端返回并做本地缓存，前端只负责展示与拦截。

Q: Vue 项目跨域常见解决方案有哪些？ ​

A:

  1. 开发环境代理（Vite/webpack devServer proxy）。
  2. 后端开启 CORS。
  3. Nginx 反向代理（生产常用）。
  4. JSONP（仅限 GET，现代场景较少）。

Q: Vue 项目部署后刷新 404 的常见原因是什么？ ​

A:

常见于 history 路由模式：服务端找不到真实文件路径。

解决方案：

  1. 服务端将未知路径回退到 index.html。
  2. 静态资源 base/publicPath 配置与部署路径一致。
  3. 检查 Nginx 或 CDN 的路由重写规则。

Q: Vue 项目如何做错误处理与监控？ ​

A:

  1. 运行时：app.config.errorHandler 统一捕获组件错误。
  2. Promise：全局监听未处理拒绝。
  3. 网络层：请求拦截器统一处理状态码与重试。
  4. 监控平台：接入 Sentry/自建日志平台，记录错误上下文与用户行为。
